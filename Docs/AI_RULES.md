# AI Development Rules

Always:

Plan before coding.

Use modular architecture.

Keep components under 300 lines.

Separate UI and logic.

Create reusable components.

Never:

Hardcode parameters.

Mix prompt logic and UI.

Skip error handling.

Assume missing requirements.

Optimization Rules:

Always use structured prompt format.

Always perform multipass optimization.

Always include verification instructions.

Always improve clarity.

Quality Rules:

Code must be production quality.

UI must remain consistent.

Parameters must always be editable.
